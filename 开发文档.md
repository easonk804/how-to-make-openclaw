 # How-to-Make-OpenClaw 开发文档

 本项目是教学项目，不追求“生产级复杂度”，追求“机制可理解、步骤可复现、测试可回归”。

 ## 1. 开发原则

 1. **每章只讲一个机制**（控制变量）。
 2. **每章三段式**：`v1` 最小可跑、`v2` 引入关键改进、`v3` 接近真实架构。
 3. **默认离线可运行**：不依赖真实 API Key。
 4. **测试先行**：每章要有章节测试，根目录要有结构和集成测试。

 ## 2. 目录约定

 ```text
 chapters/XX_topic/
 ├── README.md
 ├── exercises.md
 ├── v1_*.py
 ├── v2_*.py
 ├── v3_*.py
 └── test_chapterXX.py
 ```

 根目录固定文件：
 - `tests/test_chapter_index.py`
 - `tests/test_chapter_scaffolds.py`
 - `tests/test_final_integration.py`
 - `final/openclaw_full.py`

 ## 3. 章节开发模板

 每章 README 建议包含：
 1. 问题引入（为什么需要这个机制）
 2. v1/v2/v3 演进点（每版改了什么）
 3. ASCII 流程图（帮助初学者建立心智模型）
 4. 运行命令

 文档导航建议保持同步：
 - `docs/chapter_diagrams.md`：章节事件图（含失败/回退路径）
 - `docs/expected_outputs.md`：输出结构契约
 - `docs/final_architecture.md`：教学到 openbot-main 的架构映射
 - `docs/deployment_guide.md`：部署与验收手册

 每章 exercises 建议包含：
 - Basic（1~2 题）
 - Intermediate（1~2 题）
 - Challenge（1 题）

 ## 4. 代码风格

 - Python 3.10+
 - 函数命名尽量语义化（`run_with_lanes`、`route_session`）
 - 尽量使用小函数 + 清晰输入输出
 - 非必要不引入第三方复杂依赖

 ## 5. 测试与验证

 ### 5.1 全量测试

 ```bash
 python -m pytest -q
 ```

 ### 5.2 运行最终集成 Demo

 ```bash
 python final/openclaw_full.py
 ```

 ### 5.3 运行全部章节脚本（可选）

 ```bash
 python -c "import glob,runpy; [runpy.run_path(f, run_name='__main__') for f in sorted(glob.glob('chapters/*/v*.py'))]"
 ```

 ### 5.4 发布前部署验收（建议）

 按 `docs/deployment_guide.md` 完成以下检查：
 1. `python -m pytest -q`
 2. `python final/openclaw_full.py`
 3. 文档链接与输出契约一致性检查
 4. 敏感信息检查（`.env`、token、API Key）

 ## 6. 变更提交流程建议

 1. 先补测试或同步更新测试
 2. 再改章节代码
 3. 本地跑 `pytest -q`
 4. 更新 `docs/expected_outputs.md`（若输出结构变化）
 5. 更新 `docs/chapter_diagrams.md`（若流程/分支变化）
 6. 更新 `docs/roadmap.md`
